🔷 1. إنشاء الجدول الداخلي (Internal Table)

تم إنشاء جدول داخلي باسم customer_internal ليكون تحت الإدارة المباشرة لـ Hive، حيث يتولى النظام مسؤولية تنظيم البيانات وتخزينها في المستودع الافتراضي.

CREATE TABLE customer_internal (
    CustomerID INT,
    Name STRING,
    Email STRING,
    Phone_Number STRING,
    Address STRING,
    JOIN_Date STRING
)
ROW FORMAT SERDE 'org.apache.hadoop.hive.serde2.OpenCSVSerde'
WITH SERDEPROPERTIES (
    "separatorChar" = ",",
    "quoteChar" = "\"",
    "escapeChar" = "\\"
)
STORED AS TEXTFILE;
✅ النتيجة: تم إنشاء الجدول الداخلي بنجاح في 1.706 ثانية.
📸 لقطة شاشة: إنشاء جدول customer_internal
🔷 2. إنشاء الجدول الخارجي (External Table)

تم إنشاء جدول خارجي باسم customer_external مع تحديد موقع تخزين مخصص في HDFS، والهدف هو فصل هيكل الجدول عن الملفات الخام مما يضمن بقاء البيانات حتى بعد حذف الجدول.

CREATE EXTERNAL TABLE customer_external (
    CustomerID INT,
    Name STRING,
    Email STRING,
    Phone_Number STRING,
    Address STRING,
    JOIN_Date STRING
)
ROW FORMAT SERDE 'org.apache.hadoop.hive.serde2.OpenCSVSerde'
WITH SERDEPROPERTIES (
    "separatorChar" = ",",
    "quoteChar" = "\"",
    "escapeChar" = "\\"
)
STORED AS TEXTFILE
LOCATION '/user/hive/external/customer_external';
✅ النتيجة: تم إنشاء الجدول الخارجي بنجاح في 0.101 ثانية.
📸 لقطة شاشة: إنشاء جدول customer_external

🔷 4. حل مشكلة الفاصلة داخل عمود Address

بعض قيم عمود Address تحتوي على فواصل داخل النص، مما يفسد قراءة ملف CSV.

📖 OpenCSVSerde - المترجم المتخصص

هو القارئ المتخصص الذي تم استخدامه بدلاً من القارئ الافتراضي، ويتميز بقدرته العالية على معالجة ملفات CSV التي تحتوي على تداخل في الرموز أو تعقيدات في النصوص الطويلة مثل العناوين.

🔹 separatorChar - فاصل الحقول

يحدد الرمز المستخدم للفصل بين عمود وآخر داخل الملف، وتم ضبطه على الفاصلة (,) لتطابق بنية ملفات CSV.

🔹 quoteChar - رمز الاقتباس

يخبر المحرك بأن أي نص يقع بين علامتي تنصيص ("") يجب معالجته ككتلة واحدة، وبالتالي يتم تجاهل أي فاصلة موجودة داخل هذه العلامات.

🔹 escapeChar - رمز الهروب

يستخدم رمز الخط المائل الخلفي (\) لتعريف النظام بأن الرمز الذي يليه مباشرة هو نص عادي وليس له دلالة برمجية.

✅ النتيجة: تمت قراءة العناوين بشكل صحيح، وظهرت البيانات في الاستعلامات بدون أخطاء.
🔷 5. تجربة حذف الجداول (DROP TABLE)
DROP TABLE customer_internal;
DROP TABLE customer_external;
📸 لقطة شاشة: حذف الجدولين الداخلي والخارجي
📌 نتيجة حذف الجدول الداخلي (customer_internal):
تم حذف هيكل الجدول من قاعدة البيانات، وتم حذف الملفات الفعلية من HDFS. البيانات ضاعت بالكامل.
📌 نتيجة حذف الجدول الخارجي (customer_external):
تم حذف تعريف الجدول فقط، بينما بقيت البيانات الفعلية محفوظة في مسارها المحدد داخل HDFS.
نوع الجدولحذف الهيكلحذف البيانات

🔷 9. تحديث SCD2 (بدون UPDATE/DELETE)

بما أن Hive لا يدعم عمليات UPDATE أو DELETE، تم استخدام أسلوب INSERT فقط لتطبيق SCD Type 2 من خلال أربعة مسارات منطقية:

📌 المسار الأول: الحفاظ على السجلات التاريخية

يتم استرجاع كافة السجلات التي انتهت صالحيتها سابقاً (Is_Current = '0') لضمان بقاء السجلات غير النشطة محفوظة في النظام.

📌 المسار الثاني: الحفاظ على السجلات الحالية غير المتغيرة

يتم البحث عن العملاء الذين لم تطرأ أي تعديلات على بياناتهم، وإعادة إدراجهم كما هم للحفاظ على استمرارية البيانات التي لم تتأثر بالتحديث.

📌 المسار الثالث: إغلاق السجل القديم للمعدل

عند اكتشاف اختلاف في بيانات عميل موجود مسبقاً، يتم ضبط تاريخ الانتهاء (End_Date) على تاريخ اليوم الحالي وتغيير Is_Current إلى '0'.

📌 المسار الرابع: إدراج البيانات الجديدة والنسخ المحدثة

يتم إضافة العملاء الجدد الذين يظهرون لأول مرة، وكذلك النسخ المحدثة للعملاء الذين تغيرت بياناتهم (مع Is_Current = '1').

-- إغلاق السجلات القديمة (المعدلة)
INSERT INTO customer_scd2
SELECT 
    s.CustomerID, s.Name, s.Email, s.Phone_Number, s.Address, s.JOIN_Date,
    s.Start_Date, CURRENT_DATE() AS End_Date, '0' AS Is_Current
FROM customer_scd2 s
JOIN customer_new n ON s.CustomerID = n.CustomerID
WHERE s.Is_Current = '1'
  AND (s.Name != n.Name OR s.Email != n.Email 
       OR s.Phone_Number != n.Phone_Number OR s.Address != n.Address);
